1. MODULE D’ANALYSE PRÉDICTIVE
📌 Objectif :

Détecter automatiquement des signaux faibles ou forts de menace à partir des documents de renseignement, et fournir des scores décisionnels.
🔍 Détails :

    Extraction d’entités (personnes, groupes armés, lieux, armements) à l’aide de modèles NER (spaCy, BERT-NER, etc.).

    Analyse d’intention : détection de termes indiquant un passage à l’acte (« attaque imminente », « transport d’armes »).

    Score de menace : probabilité qu’un message soit associé à une menace réelle.

    Score de crédibilité : basé sur l'historique de la source et la cohérence avec d’autres documents.

    KPI calculés :

        score_menace (entre 0 et 1)

        delta_score (variation du score par rapport au temps)

        indice_de_confiance

        fréquence_d’apparition

2. SUIVI DU RENFORCEMENT / AFFAIBLISSEMENT DES PRÉDICTIONS
📌 Objectif :

Suivre l’évolution de la confiance dans une prédiction au fil du temps et des documents.
🔍 Détails :

    Calcul de variation (ΔScore) : si un document fait passer un score de 0.62 à 0.81 → renforcement.

    Analyse graphe : si l’entité devient plus connectée dans le graphe relationnel, cela indique une montée en importance.

    Visualisation KPI :

        Graphes temporels

        Tableaux de variation

        Représentation visuelle du graphe d’acteurs

3. MODULE D’ANALYSE PRESCRIPTIVE
📌 Objectif :

Proposer automatiquement des actions concrètes à partir d’une prédiction.
🔍 Détails :

    Prescriptions de renforcement (si la prédiction est crédible) :

        Activer la surveillance (SIGINT, HUMINT)

        Demander un recoupement inter-agence

        Ajouter de nouvelles entités au graphe

    Prescriptions d’affaiblissement (si incohérences détectées) :

        Baisser la pondération de la source

        Demander confirmation terrain

        Geler le suivi jusqu'à validation

    LLM peut générer ces prescriptions avec justification textuelle :

        « Compte tenu de la mention d'une opération coordonnée et de deux sources distinctes, nous recommandons de mobiliser un observateur terrain et d’intensifier la veille SIGINT. »

4. INTÉGRATION DES LLM POUR UNE ANALYSE AVANCÉE
📌 Objectif :

Utiliser des modèles de langage (LLM) pour une compréhension contextuelle et stratégique.
🔍 Détails :

    Amélioration NER & Sentiment : extraction d’acteurs indirectement nommés.

    Analyse de contradictions : comparer automatiquement deux documents pour détecter une incohérence.

    Génération de résumés ou explications de score :

        Pourquoi ce document augmente le risque ?

        Quelles entités ont changé de centralité ?

    Fonctionne en mode sécurisé : LLM local (Ollama, GGUF) pour les zones classifiées.

5. MODULE DE SCÉNARIOS MANUELS
📌 Objectif :

Permettre aux analystes de définir un événement projeté à surveiller, basé sur les KPI.
🔍 Détails :

    Exemple :
    Scénario : attaque à Tombouctou dans les 7 prochains jours si score_menace > 0.7 + présence du groupe XYZ + mentions de "coordination".

    Composants d’un scénario :

        Conditions (scores, entités, mots-clés)

        Temporalité (durée de validité)

        Lieu ou contexte

    Suivi automatique :

        À chaque nouveau document, les conditions sont testées.

        État : Non déclenché / Partiellement réalisé / Réalisé / Invalide

6. MODULE D’AUTO-APPRENTISSAGE & VALIDATION HUMAINE
📌 Objectif :

Faire évoluer le modèle à partir de l’expérience.
🔍 Détails :

    Auto-apprentissage :

        Si une prédiction s’est avérée correcte, on renforce les patterns associés.

        Si elle a échoué, le modèle apprend à les éviter (faux positifs).

    Validation humaine :

        Une interface permet à l’utilisateur de dire : “Ce scénario s’est bien produit” ou “non”.

        Retour utilisé pour ajuster le classifieur ou les pondérations.

7. GÉNÉRATION AUTOMATISÉE DE REQUÊTES DE COLLECTE
📌 Objectif :

Déclencher automatiquement une demande de renseignement si l’information est incomplète.
🔍 Détails :

    Conditions de génération :

        Score élevé mais manque de données

        Contradictions entre documents

        Incohérence ou impossibilité de vérifier

    Requête générée :

        Objectif clair (ex: “Vérifier la présence d’éléments armés dans le secteur Est”)

        Type de collecte souhaitée (HUMINT, IMINT, SIGINT)

        Niveau d’urgence

    Anti-boucle logique :

        Une seule requête par scénario

        Ne se répète pas si déjà satisfaite ou expirée
